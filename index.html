Developer Briefing — June 2026

Sextech CRM Platform
Technical Review

An independent escort and dominatrix CRM built as a multi-tenant SaaS. This hub contains the strategy report, interactive schema diagrams, and a sample data export.

Context: Two parallel builds exist — one built with Claude (Next.js + Supabase + Vercel, ~40hrs) and one built on Lovable (also Supabase-backed). The goal is to merge the better UI of the Lovable build with the stronger infrastructure of the custom build. Advice is sought on architecture, WhatsApp integration strategy, and security.

Tech stack

Custom build: Next.js 16 · TypeScript · Supabase (PostgreSQL + Auth + RLS) · Vercel · Baileys (WhatsApp QR bridge)

Lovable build: Lovable (React) · Supabase · Vercel

Key constraints:

  • Meta/WhatsApp is a platform risk — official API creates visibility with Meta, which has deplatformed sextech firms
  • Stripe and PayPal typically refuse sex work adjacent businesses
  • GDPR applies — users are in France and Switzerland, data is sensitive
  • FOSTA-SESTA creates US payment processor exposure
  • WhatsApp integration must be seamless for end users from day one (non-negotiable requirement)
