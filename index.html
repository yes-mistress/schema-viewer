Developer Briefing — June 2026

Sextech CRM Platform — Technical Review

A multi-tenant SaaS CRM built for independent sex workers — escorts and dominatrices. This hub contains the strategy report, interactive schema diagrams, and anonymised data exports from both builds.

Context

Two parallel builds exist and need to be merged:

  • Clare's build — custom Next.js 16 + TypeScript + Supabase + Vercel (~40hrs). Stronger infrastructure: WhatsApp bridge (Baileys), multi-tenant RLS, automations, flows, practices system, lead form API.
  • Josephine's build — Lovable (React) + Supabase. Superior UI: better dashboard, pipeline forecasting, safety check-in workflow, richer contact fields.

Key constraints: Meta visibility risk for sextech, Stripe and PayPal typically refuse the industry, GDPR applies (France + Switzerland users), WhatsApp must work seamlessly from day one.
